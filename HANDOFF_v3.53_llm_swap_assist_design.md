## 🎯 โจทย์
"โอกาสในการใช้ LLM ในการแลกเวร โดยให้ user ยืนยันอีกครั้งก่อนแลกเวรจริง"
→ อยากให้พิมพ์บอกเป็นภาษาพูด (เช่น "แลกเวรวันที่ 20 กับตี๋") แทนกดเลือกทีละขั้น แต่ **AI ห้ามยิง
แลกเวรเองทันที** ต้องมีจุดให้ user เห็นสิ่งที่ AI เข้าใจแล้วกดยืนยันก่อนเสมอ

## ✅ แนวทางที่เสนอ — "AI แค่เติมฟอร์ม ไม่ใช่คนกดยืนยัน"
ใช้ **flow แลกเวรเดิมทั้งหมด** (picker → `_lwwTransferConfirm` → `swfOpenConfirm` → เผยแพร่ —
ดู v3.48) เป็นตัวบังคับยืนยันสุดท้ายอยู่แล้ว **ไม่แตะ logic การยิง overlay จริงเลย**

เพิ่มแค่ **ทางลัดก่อนหน้า picker**:

```
[ผู้ใช้พิมพ์เป็นคำพูด]
        ↓
llmParseSwapRequest(text, myShiftsThisMonth)   ← เรียก Gemini ฝั่ง server
        ↓
[dialog แสดงสิ่งที่ AI เข้าใจ] "คุณหมายถึง: สลับเวร 20 ก.ค. (เช้า NM5) กับ ตี๋(ช) ใช่ไหม?"
   ⚠️ นี่คือ "ยืนยันครั้งที่ 1" — เฉพาะยืนยันว่า AI ฟังถูก ไม่ใช่ยืนยันแลกเวร
   [✅ ใช่ ถูกต้อง]  [✏️ พิมพ์ใหม่]  [❌ ไม่ใช่ เลือกเองแทน]
        ↓ (กด "ใช่")
prefill เข้า picker เดิม (เหมือนกดเลือกเวรเองทุกประการ)
        ↓
_lwwTransferConfirm → ไดอะแกรม A→B→C (v3.48) → "ยืนยันครั้งที่ 2" (ของเดิม, บังคับอยู่แล้ว)
        ↓
กด "เผยแพร่" → ยิงแลกเวรจริง
```

**หลักการ:** AI เป็นแค่ตัวแปลภาษาพูด → โครงสร้าง ไม่มีสิทธิ์เขียนอะไรลง Firebase เอง
ทุก path (พิมพ์บอก AI หรือกดเลือกเอง) ต้องผ่านจุดยืนยันสุดท้ายเดียวกัน — **ไม่มีทางลัดข้าม
การยืนยันได้เลย แม้ AI จะมั่นใจ 100%**

### กันเคสมั่ว/AI เข้าใจผิด
- ถ้า `confidence` ต่ำ หรือ parse ไม่ออก (เช่น ชื่อคนไม่อยู่ในตาราง, วันที่ไม่มีเวร) → **ไม่เดา** ขึ้น
  ข้อความ "ฟังไม่ออก ลองพิมพ์ใหม่ หรือเลือกเองด้านล่าง" ตกไปที่ picker มือเปล่าทันที
- Prompt ผูก grounding เป็นรายการเวรของ "ตัวเอง" เดือนนั้นเท่านั้น (ไม่ยิง whole roster ทุกคนออกไป
  นอกระบบโดยไม่จำเป็น) — ส่งเฉพาะ shift ที่เกี่ยวข้องกับ query
- บังคับ Gemini ตอบเป็น JSON schema (structured output) กันข้อความหลุด/prompt injection จากช่อง
  พิมพ์อิสระ

## 🔧 จุดที่ต้องแก้ (ถ้า approve)
1. ไฟล์ใหม่ `Phase_LLM_SwapAssist.js`
   - `llmParseSwapRequest_(text, monthId)` — ดึงเวรของ user จาก session, สร้าง prompt, เรียก
     Gemini REST (`UrlFetchApp`), บังคับ `responseSchema` แบบ JSON, คืน
     `{ok, confidence, action, myShiftId, targetPersonName, targetShiftId, humanSummary}`
   - key เก็บที่ `PropertiesService.getScriptProperties().getProperty('GEMINI_API_KEY')`
     (pattern เดียวกับ `LINE_CHANNEL_ACCESS_TOKEN` ใน `Phase_F2_LINE.js:21`) — **ไม่ hardcode**
   - ไม่มี key → ฟีเจอร์นี้ปิดเงียบ (ปุ่ม "พิมพ์บอก AI" ไม่โผล่), flow มือเปล่าเดิมใช้ได้ปกติ
2. `Index.html` — เพิ่มปุ่ม "💬 พิมพ์บอก" ในหน้าเมนู `phxViewSwap` (ข้างปุ่ม 4 ปุ่มเดิม จาก v3.48)
   - textarea + ปุ่มส่ง → เรียก `google.script.run.llmParseSwapRequest_`
   - dialog ยืนยันครั้งที่ 1 (ใหม่) → เรียก `confirmPickShift`/entry point เดิมของ picker ด้วยค่า
     prefill (ต้องดูโค้ดจริงว่า `confirmPickShift` รับ prefill param ได้ไหม หรือต้อง refactor เล็ก)
3. ไม่แตะ `_lwwTransferConfirm`, `swfOpenConfirm`, overlay-write functions ใดๆ ทั้งสิ้น
